4. 이름 기반 로직 처리
4.1. 데이터 소스(Source) 기반 스탯 독립 곱연산

엔진은 데이터 파일명(또는 시트명)을 스탯 연산의 출처(Source) 식별자로 자동 인식합니다.

규칙:

  • 동일 시트(파일) 내의 같은 스탯 → 합산
  • 서로 다른 시트(파일) 간의 같은 스탯 → 독립 곱연산

예시:

펫 시트에서 dmg_p 30% → 펫_dmg_p 버킷 탈것 시트에서 dmg_p 30% → 탈것_dmg_p 버킷 최종: 1.3(펫) × 1.3(탈것) = 1.69배 (단순 합산 1.6보다 높은 효율)

파일명 형식: Source_ContentName (예: Pet_Effect.csv → Source: Pet)

※ 실제 프로젝트 데이터 보호를 위해 파일명·시트명·콘텐츠명을 가상의 예시로 재구성했습니다.

    planning/ 6명(문서경찰 제외)은 Monday.com 권한이 없다. Monday.com 변경(이슈 생성·상태 전환·이관 등)이 필요하면 PM에게 요청한다. PM은 요청 내용을 정리해 사용자에게 전달하고, 사용자가 직접 처리한다.

    1. 사용자 요청 수신 2. 작업 범위 식별 → planning/ 트리 게이팅 계획 수립 3. 계획 보고 → 승인 후 진행 4. 각 담당자에게 위임 → 결과 취합 5. dev/ 영향 사안 발견 시 사용자에게 보고 (직접 위임 금지, CSVManager CSV 동기화만 예외)
